## Release: v0.6.0 - 2026-03-13

**Status**: Documentation prepared and ready for release

### Key Features Documented

1. **Dead Time Learning** (Issue #62)
   - Automatic dead time calculation for more accurate predictions
   - New sensor: `sensor.intelligent_heating_pilot_<device>_dead_time`
   - Updated prediction formula: includes dead time before temperature rise calculation
   - Documented in: HOW_IT_WORKS.md, CONFIGURATION.md, USER_GUIDE.md

2. **Zero-Retention Mode**
   - Set `lhs_retention_days=0` to disable LHS history storage
   - Useful for minimal deployments or testing
   - Documented in: CONFIGURATION.md, CHANGELOG.md

3. **Startup Performance Improvements**
   - Async incremental recorder loading
   - FIFO RecorderAccessQueue for multi-device safety
   - Smart event filtering to reduce log spam
   - Contextual LHS lazy loading (only load current hour at startup)
   - Documented in: CHANGELOG.md (Fixed section)

4. **Timer-Based Anticipation Triggering** (Issue #84)
   - Replaced event-driven with timer-based triggering
   - Improved reliability of preheating system
   - Documented in: CHANGELOG.md (Changed section)

5. **Maintained Features from Previous Releases**
   - IHP Enable/Disable Switch (v0.5.0)
   - Optional Scheduler Support (v0.5.0)
   - Fixed Configuration Values Being Ignored When Falsy (v0.5.0)
   - No-scheduler KeyError spam fix (v0.5.0)
